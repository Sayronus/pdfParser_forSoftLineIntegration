Здравствуйте. В данном репозитории наша команда предоставляет программу для парсинга файлов спецификации и создания таблицы на их основе.
Ниже прилагаю пару комментаривает по ее использованию.

1. Перед запуком главного скрипта "program.py" убедитесь в наличии установленного Python на компьютере и некоторых нужных библиотек:
  camelot-py, pandas, openpyxl, pdfplumber, ghostscript и tkinter. Если хотя бы одной из них нет, нужно провести установку: в оболочке Python прописать консольную команду -
                pip install [название библиотеки].
  Можно выполнить одну-единственную команду, проверяющую наличие всех нужных библиотек и установливающую недостающие:
                pip install camelot-py pandas openpyxl pdfplumber ghostscript tkinter

  Также, после установки директории на свой компьютер, удалите файл DELETEME в папке "output_pdf" - он является простой заглушкой для хранения этой папки в git.

2. Перед запуском программы нужно указать папку инпута исходных pdf-файлов: по умолчанию такая папка должна находится в той же директории, что и сам скрипт "program.py",
  и называться "pdf_input". Обратите внимание, что путь до папки с файлами, которые нужно кинуть на парсинг,можно изменить, прописав нужный путь
  в программе - за это отвечает переменная "INPUT_DIR". 
  В папку инпута должны храниться данные для преобразования - исходные pdf-файлы, по которым и создается сводная таблица.

3. Во время выполнения программы в консоль выводится читаемый на данный момент файл - так можно отслеживать прогресс выполнения.
  При завершении программы в директории с главным скриптом появится xlsx-таблица по заданному формату - по умолчанию это файл под названием "result.txt",
  аутпут можно также настроить, за это отвечает переменная "OUTPUT_EXCEL" - в которой и находится вся сведенная информация.
  (формат таблицы можно также тонко настроить в переменных программы - все они декларируются в начале или в соответствующих процессам функциях)

-----------------------------------------------------------------------------------------------------------------------

Для тестирования можно использовать файлы из папки "output_pdf_example": просто перекиньте файлы из этой папки в папку "output_pdf". На всех файлах из этой папки программа
завершается примерно за 1500 секунд.
